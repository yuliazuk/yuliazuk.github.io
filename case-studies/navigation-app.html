Calling app: oncall navigation on Android

Project

Design the in-app navigation for our telephony app on Android, addressing the challenge of inconsistent navigation patterns across different devices. Through user testing and iterative design, we developed a simplified and intuitive navigation solution. This improved user experience and satisfaction, providing a seamless experience for our business clients.

Goal

Improve user navigation within the app while on a call on Android devices.

Challenges

  • Variability in Navigation Patterns: Unlike iOS, where a standardized navigation pattern exists, Android devices have scientifically different solutions due to custom interfaces created by different phone manufacturers. This variability posed a challenge in ensuring a uniform user experience.
  • User Experience Diversity: Users had varying levels of familiarity with different navigation patterns. Some users found certain native options familiar and easy to use, while others struggled, finding these options entirely new and difficult to navigate.
  • Technical Constraints: Our developers faced difficulties in controlling and mimicking the native navigation patterns across different Android devices, making it hard to provide a consistent experience.

Team

  • UX/UI designer
  • Project Manager
  • Product Manager
  • Mobile Developer

My role

As the UX designer for the project, I was responsible for researching and analysing user behaviour, designing and iterating navigation patterns, and ensuring a seamless and intuitive navigation experience within the telephony app while users were on a call.

Initial approach

Research

  • Compared native navigation patterns on different Android devices.
  • Reviewed iOS navigation patterns for reference.

Implementation

  • Tried various native Android navigation options.
    • Icon in the status bar
    • Fab button
  • Conducted unmoderated user testing to gather feedback on each option on usertesting.com

Findings

While some users were only familiar with the first native navigation option and others with the second, all users struggled significantly when trying to use an option that was not familiar to them.
This highlighted the need for a more universal and intuitive navigation solution.

Solution Iteration Process:

  1. User Feedback Analysis: We thoroughly analyzed the feedback to understand why certain navigation patterns were problematic and identified common user pain points.
  2. Brainstorming Sessions: Through collaborative sessions with developers and designers, we explored alternative solutions that prioritized user-centric design principles and cross-platform consistency.

Results

The new navigation pattern was tested with a diverse group of users, and the feedback was overwhelmingly positive. Users found the new navigation intuitive and easy to use, leading to high satisfaction and successful task completion rates. The solution effectively improved the user experience across various Android devices, meeting the project's primary objective.

Lessons Learned:

The importance of user-centric design and thorough testing became evident.
Flexibility and adaptability in design are crucial for apps with a diverse user base and varying experiences.

Future Recommendations:

Continuously monitor user feedback to identify areas for ongoing improvements.
Explore potential areas for further research or development to refine the navigation experience further.
